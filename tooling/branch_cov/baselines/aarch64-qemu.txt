# branch-cov baseline — aarch64/qemu tracer bullet
#
# Each line: <hex-addr> <taken|not_taken>. These are (branch, outcome)
# pairs that the single-vCPU tracer-bullet boot legitimately cannot
# exercise. Any DELTA from this baseline — new gap or closed gap —
# fails the run. Tighten this file when:
#   - new tests cover a branch path (drop the entry)
#   - boot.S grows new branches (add the entries once we've decided
#     they're intentionally uncovered)
#
# Regenerate by running the tracer bullet under TRACE=1 and reading
# the "Gaps:" list out of branch-cov's output.
#
# Address invariant: every offset below is relative to the ELF's load
# address (0x0 in the linker script + 0x80000 text_offset added by
# branch-cov via --load-offset). Any change that re-orders / re-sizes
# the .text contributors (boot.S, memory/bytecode_vm.S,
# memory/allocator.S, memory/init_memory_layout.S) shifts every
# address in this file. Regenerate wholesale rather than trying to
# patch in place.

# Regen history:
#   2026-05-13: allocator wire-up landed (init_memory_layout calls
#     memlayout_run_allocator; aarch64/qemu's regions.yaml is the
#     empty stub per D066 Q-F "QEMU cells get NOTHING"). The bytecode
#     VM and allocator bodies are now reachable BUT iterate a zero-
#     length .memreq range, so the per-record branches (size_bc /
#     align_bc eval, bump arithmetic, lifetime dispatch) stay
#     uncovered at tracer-bullet runtime. Diff-harness coverage in
#     tooling/tests/test_memlayout_*_diff.py exercises every opcode
#     + two-pass allocator path under user-mode QEMU; these gaps
#     reflect that the kernel-side tracer doesn't visit them.
