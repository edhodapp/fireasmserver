# branch-cov baseline — aarch64/qemu tracer bullet
#
# Each line: <hex-addr> <taken|not_taken>. These are (branch, outcome)
# pairs that the single-vCPU tracer-bullet boot legitimately cannot
# exercise. Any DELTA from this baseline — new gap or closed gap —
# fails the run. Tighten this file when:
#   - new tests cover a branch path (drop the entry)
#   - boot.S grows new branches (add the entries once we've decided
#     they're intentionally uncovered)
#
# Regenerate by running the tracer bullet under TRACE=1 and reading
# the "Gaps:" list out of branch-cov's output.
#
# Address invariant: every offset below is relative to the ELF's load
# address (0x0 in the linker script + 0x80000 text_offset added by
# branch-cov via --load-offset). Any change that re-orders / re-sizes
# the .text contributors (boot.S, memory/bytecode_vm.S,
# memory/allocator.S, memory/init_memory_layout.S) shifts every
# address in this file. Regenerate wholesale rather than trying to
# patch in place.

# ---- boot.S: dead branches in the always-present code paths ----
#
# cbnz x5, .Ldone — secondary-CPU entry. vcpu_count=1 in our config,
#   so the primary always falls through. Only "taken" is missing.
# cbz w2, .Lemit_done — emit_bytes early-return for empty buffers.
#   All call sites pass a non-empty buffer, so only "taken" is missing.
# tbnz w3, #5, .Lemit_loop — wait-while-FIFO-full. QEMU's PL011
#   emulation drains instantly, so TXFF==1 never observed.
#   Only "taken" is missing.

# ---- memlayout VM + allocator dead-code branches -------------------
#
# arch/aarch64/memory/bytecode_vm.S (D060 step 3A) and
# arch/aarch64/memory/allocator.S (step 3B) are linked into the
# kernel image and reachable via D060 step 4.2's wire-up
# (boot.S → init_memory_layout). However init_memory_layout is
# still a return-immediately stub (.memreq section is empty until
# step 5 declares regions), so the bytecode VM and allocator
# bodies stay dead at tracer-bullet runtime. Will close in step 5
# once a module declares a region. The ratchet will then notice
# the closed gaps, fail, and force a re-baseline — exactly the
# mechanism the ratchet is designed to provide.
#
# Out-of-band coverage of the same code happens in
# tooling/tests/test_memlayout_diff.py and test_memlayout_alloc_
# diff.py, which exercise every opcode + every two-pass allocator
# path under user-mode QEMU.
